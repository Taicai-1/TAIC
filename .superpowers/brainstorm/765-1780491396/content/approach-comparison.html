Approche pour le rendu Markdown

3 niveaux d'investissement — quel style veux-tu pour les réponses IA ?

A

Clean & Minimal (Recommandé)

Un composant MarkdownRenderer partagé avec tous les éléments bien stylés : titres, listes, tables, blockquotes, code blocks avec fond gris. Pas de librairie externe supplémentaire. Rendu propre, cohérent entre agent et team chat.

Avantages

  • Aucune dépendance ajoutée
  • Cohérence agent/team/contributions
  • Rapide à implémenter
  • Léger en bundle

Limites

  • Pas de coloration syntaxique dans les blocs de code
  • Style manuel pour chaque élément
B

Premium avec Syntax Highlighting

Comme A, mais avec react-syntax-highlighter (ou rehype-highlight) pour coloriser les blocs de code. Style inspiré de ChatGPT/Notion — titres bien marqués, blockquotes élégantes, tables zebra-striped.

Avantages

  • Rendu professionnel/premium
  • Code colorisé = lisibilité ++
  • Style complet pour tous les éléments

Limites

  • +30-50KB de bundle (highlight.js)
  • Une dépendance de plus à maintenir
C

Tailwind Typography (prose)

Utiliser @tailwindcss/typography (plugin prose) au lieu de styler chaque élément manuellement. Le plugin gère tous les éléments HTML automatiquement avec un style éditorial cohérent.

Avantages

  • Tous les éléments stylés automatiquement
  • Très peu de CSS custom
  • Bien maintenu par Tailwind Labs

Limites

  • Moins de contrôle fin sur chaque élément
  • Besoin d'overrides pour les couleurs user/agent
  • Pas de syntax highlighting natif

💡 Les options B et C sont combinables : prose pour la base + syntax highlighter pour le code.
